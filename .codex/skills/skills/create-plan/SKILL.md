# Plan guidelines

## Rules
- Every plan must begin with a small summary of the goal of the plan
- The plan steps must be small, autoconsistent, testable and progressive.
- Each steps must contain a completed flag "[]" that will be updated during the implementation of the plan
- The last 2 steps of each plan, should be:
    1. One dedicated to update all the documentation files adding/updating/deleting the appropriate section in order to keep them up to date
    2. And the other to update/change/delete all the appropriate tests in order to reflect the new implementation
